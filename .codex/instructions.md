# Codex Instructions

## Operational Principles

- Never commit directly to main.
- Always use feature branches.
- Prefer reusable modular systems.
- Use clean architecture.
- Favor Tailwind and shadcn/ui patterns.
- Optimize for mobile-first experiences.
- Preserve cinematic luxury hospitality tone.
- Avoid visual clutter.
- Prioritize analytics instrumentation and accessibility.
- Create clean PR summaries.

## Working Style

Every contribution should improve the repository as an operating system, not just as a collection of files. Keep documentation clear, architectural decisions explicit, and future implementation paths easy to follow.
