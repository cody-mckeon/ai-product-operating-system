# Codex Workflow

## Default Workflow

1. Start from a feature branch.
2. Review the relevant documentation and existing patterns.
3. Keep changes focused on the requested scope.
4. Favor reusable systems over one-off decisions.
5. Document architectural intent when introducing new patterns.
6. Verify accessibility, analytics, and mobile-first implications.
7. Prepare a clear pull request summary with risks and follow-ups.

## Review Expectations

Pull requests should explain the problem, the approach, the files changed, and the impact on the broader product operating system.
