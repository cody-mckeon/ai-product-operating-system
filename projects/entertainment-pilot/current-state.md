## Primary User Journeys Observed

### Entertainment Discovery

1. User lands on the Entertainment page.
2. User sees property navigation and a large promotional hero carousel.
3. User can move through hero slides using left/right arrows.
4. User can use the anchor bar to jump to theatre, Ayu Dayclub, or Zouk Nightclub sections.
5. User reads AEG intro copy that frames the entertainment offering.

### Event Browsing And Selection

1. User reaches the event lineup after the intro section.
2. User can constrain or change browsing using date range, category filters, and view icons on desktop.
3. User scans event cards by artwork, event title, venue badge, and date.
4. User chooses between event-level CTAs: tickets, booking, tables, or learn-more actions depending on the card.
5. User can request additional results with `LOAD MORE`.

### Venue Orientation

1. User reviews the Resorts World Theatre section with venue capacity/positioning copy and partner logos.
2. User reviews Theatre Amenities content and carousel imagery.
3. User reviews Venues content for Ayu Dayclub and Zouk Nightclub.
4. User can use venue-level `BOOK NOW` CTAs, though resulting destinations are not visible in the provided evidence.

### FAQ And Support

1. User reaches FAQ tabs after the theatre/amenity content.
2. User can switch among FAQ categories or use previous/next category controls.
3. User can select question rows to reveal answers, though expanded answers are not shown.
4. User can continue to footer navigation, phone contacts, or the floating chat button for support.

## Current-State Takeaway

The current Entertainment page is a single long-form landing and browsing experience. It combines marketing orientation, event discovery, commercial CTAs for tickets/tables/booking, venue education, FAQ support, and property footer navigation in one scroll. Desktop emphasizes a large hero and four-column event grid, while mobile condenses navigation into a sticky header and presents event cards as large touch-oriented rows. The provided evidence supports documenting page/module structure and visible CTA patterns, but not downstream booking, ticketing, table, menu, FAQ-expanded, or checkout behavior.
