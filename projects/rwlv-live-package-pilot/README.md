# RWLV Live Package Pilot

## Project Overview

This project is the first real-world proving ground for the AI Product Operating System. It focuses on the Resorts World Live experience and the opportunity to better merchandise entertainment with rooms, dining, loyalty, VIP access, and other resort offerings.

The immediate goal is not to redesign the current experience. The goal is to create a repeatable AI-assisted product workflow that can move from opportunity discovery to concept generation, concept evaluation, design prompting, telemetry planning, and implementation recommendation.

## Business Context

Resorts World Las Vegas has a broad destination ecosystem: live entertainment, hotel rooms, restaurants, nightlife, resort amenities, and loyalty relationships. The current business opportunity is to make those assets feel more connected during high-intent entertainment planning moments.

Guests considering a show, residency, or event may also be planning a weekend, choosing a room, selecting dinner, coordinating a group, or deciding whether to make the trip feel special. A stronger package architecture can reduce planning friction, increase qualified booking intent, and make the resort feel like a curated host rather than a set of disconnected offers.

## Pilot Objectives

- Identify package and merchandising opportunities in the current Resorts World Live journey.
- Generate structured concepts for entertainment-led resort packages.
- Evaluate concepts against brand, package, conversion, implementation, and analytics criteria.
- Create Figma Make and UX generation prompts that preserve doctrine alignment.
- Define telemetry needed to understand package engagement, comparison, booking intent, and conversion.
- Produce a recommendation template that supports implementation decisions without skipping product judgment.

## Success Criteria

- The pilot produces reusable documentation that can be applied to future hospitality initiatives.
- Concepts are evaluated before design generation using a consistent scorecard.
- Package ideas include audience, occasion, inclusions, constraints, CTA strategy, and measurement expectations.
- Figma prompts are specific enough to generate reviewable design drafts instead of generic luxury screens.
- Telemetry planning distinguishes attention, consideration, intent, conversion, friction, expansion, and recovery.
- Final recommendations can be reviewed by product, design, marketing, analytics, revenue, and operations stakeholders.

## Relationship To The AI Product Operating System

This project operationalizes the repository doctrine:

- Brand doctrine: composed, premium hospitality; conversion with respect; editorial clarity.
- Components doctrine: reusable story units with journey stage, action, and telemetry intent.
- Analytics framework: surfaces, modules, entities, actions, signals, funnels, experiments, and insights.
- Package architecture: packages as narrative promises with operational requirements.
- Agent definitions: package, conversion, analytics, design review, and Figma agents contribute distinct review passes.
- AI design execution workflow: opportunity intake, problem definition, analytics review, package strategy, UX concept generation, AI design exploration, and recommendation.

The project should become a reusable case study for how the operating system turns a real business opportunity into governed product work.
