# Figma Prompts

These prompts are starting points for Figma Make and AI-assisted design exploration. They should be used only after a concept has been scored and approved for visual exploration.

## Prompt Principles

- Start from the guest motivation and package promise.
- Specify journey stage, audience, primary emotion, primary action, and telemetry intent.
- Use reusable component names from the component doctrine.
- Preserve composed luxury, editorial clarity, and conversion with respect.
- Make inclusions, constraints, pricing posture, and next steps legible.
- Do not invent availability, access, savings, exclusivity, or operational guarantees.

## Figma Make Prompts

### Package Landing Page Draft

Create a mobile-first Resorts World Live package landing page concept for `[concept name]`. The experience should feel cinematic, refined, warm, calm, and commercially clear. The primary guest is `[segment]`, planning `[occasion]` around `[event/show/weekend]`.

Use these reusable modules: Experience Hero, Weekend Package Module, Stay + Event Bundle, Dining Experience Feature, Experience Timeline, Premium Offer Stack, Experience Comparison Module, Immersive CTA Section, FAQ and Policy Module, and Curated Recommendation Rail.

The page should communicate:

- The core package promise.
- What is included.
- What is optional.
- What constraints or eligibility rules must be understood.
- Why the package is better than assembling the trip manually.
- The primary CTA: `[CTA]`.
- Secondary actions: save, compare, inquire, or continue exploring.

Include visual hierarchy and annotations for telemetry intent: package impression, inclusion expansion, comparison interaction, CTA click, availability check, booking start, and abandonment risk.

### Mobile Journey Draft

Create a mobile-first flow for a guest moving from Resorts World Live event discovery to package consideration. Focus on thumb-friendly sequencing, readable package details, concise CTAs, and a persistent but respectful booking action.

Show states for:

- Event detail to package module.
- Package detail view.
- Inclusion expansion.
- Date or availability check.
- Save or compare.
- Booking or inquiry start.

Avoid dense card grids and aggressive urgency language. The flow should feel like concierge guidance.

## UX Generation Prompts

### Concept Structure Prompt

Generate three UX concepts for an entertainment-led Resorts World package. Each concept must include audience, occasion, promise, journey sequence, component candidates, CTA hierarchy, mobile behavior, measurement intent, and operational assumptions.

Compare the concepts by package strength, conversion potential, implementation effort, analytics readiness, and brand alignment.

### Journey Mapping Prompt

Map the current and proposed journey for a guest who discovers a show on Resorts World Live and considers turning it into a weekend stay. Include discovery, orientation, consideration, intent, commitment, expansion, and recovery stages.

Identify the surface, module, entity, action, signal, funnel role, likely friction, and required telemetry for each stage.

## Package Generation Prompts

### Package Strategy Prompt

Create a package strategy for `[event/show/seasonal moment]` at Resorts World Las Vegas. The package should include audience, occasion, emotional benefit, business objective, core promise, inclusions, exclusions, upgrades, eligibility, constraints, operational owner, primary CTA, comparison logic, and measurement plan.

Make the package feel curated rather than overloaded. Use value clarity, planning ease, and hospitality pacing instead of discount pressure.

### Package Variant Prompt

Generate five variants of the same entertainment-led package:

- Calm luxury.
- Effortless planning.
- Celebration weekend.
- Insider access.
- Culinary anticipation.

For each variant, define the package promise, included elements, strongest CTA, best-fit guest, measurement priority, and reason to reject or refine.

## Experimentation Prompts

### Experiment Brief Prompt

Create an experiment brief for `[concept name]`. Include guest problem, hypothesis, audience, target surface, variant rationale, primary metric, secondary metrics, guardrail metrics, required telemetry, decision rule, and operational dependencies.

The experiment should improve package comprehension, qualified intent, comparison confidence, or booking flow quality.

### CTA Experiment Prompt

Generate CTA variants for an entertainment-led package journey. Evaluate each CTA by journey stage, guest readiness, expected intent quality, downstream measurement, and brand risk.

Avoid pressure language, fake scarcity, and clickbait. Prefer language such as check availability, build your weekend, reserve this experience, compare packages, or request VIP details when appropriate.
