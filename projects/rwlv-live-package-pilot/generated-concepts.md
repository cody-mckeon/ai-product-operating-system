# Generated Concepts

All concepts below use existing components only. None requires a new package builder, new custom module, or new interaction pattern. Each concept assumes stakeholder validation for eligibility, inventory, terms, tracking, and destination URLs.

## Concept 1: Stay + Show Feature Card

### Summary

Repurpose an existing feature or promo card to present one clear room + ticket package near the top of the Resorts World Live page.

### Target Guest

Guests interested in a featured show, residency, or event weekend who may be open to staying overnight.

### Package Promise

"Turn the show into a stay" with a room path and ticket path presented as one coordinated planning option.

### Existing Components

- Feature card or promo card.
- Existing image treatment.
- Existing CTA button pair.
- Existing terms/detail link if available.

### Placement

Below the hero or immediately after a featured event module.

### CTA Strategy

- Primary: "View room + ticket package" or "Check package availability."
- Secondary: "Buy tickets only."

### Operational Assumptions

- Package can route to an existing offer, booking, or campaign URL.
- Ticket purchase may remain a separate flow if bundling is not technically available.
- Copy must state whether tickets are included or purchased separately.

### Measurement Intent

Track package impression, card engagement, CTA click, booking start, ticket-only fallback, and downstream room attachment.

### Risks

- Package language could overpromise if ticket and room inventory are not truly bundled.
- If the card appears too promotional, it may compete with ticket conversion.

## Concept 2: Package-Aware Event Cards

### Summary

Enhance eligible event cards using existing card fields: badge, metadata line, and secondary CTA.

### Target Guest

Guests scanning multiple events and deciding which show or date is worth planning around.

### Package Promise

Eligible events visibly support a stay option without forcing users out of event discovery.

### Existing Components

- Event card.
- Existing badge or label style.
- Existing secondary link or CTA pattern.
- Existing card grid or carousel.

### Placement

Within the current event listing/grid wherever eligible events appear.

### CTA Strategy

- Primary remains ticket-focused.
- Secondary: "View stay package."
- Badge: "Room package available."

### Operational Assumptions

- Eligibility can be manually configured for a curated event set.
- The same destination pattern can be reused across eligible events.
- Event card layout has space for a concise secondary message.

### Measurement Intent

Track package badge impressions, package CTA clicks by event, event-to-package detail rate, and ticket CTA impact as a guardrail.

### Risks

- Card density may become too high on mobile.
- Some events may not have package inventory, requiring clear eligibility rules.

## Concept 3: Make It A Stay Detail Card

### Summary

Add an existing offer/promo card to event detail pages near the ticket CTA, focused on turning a selected event into a room stay.

### Target Guest

Guests who have already selected a show or event and are evaluating next steps.

### Package Promise

The page helps the guest decide whether to buy only tickets or plan a complete stay around the event.

### Existing Components

- Offer card or promo card.
- Existing detail/rich text block.
- Existing CTA pair.
- Existing accordion/details block for inclusions and restrictions.

### Placement

Near the event detail CTA area, after show information and before lower-page related content.

### CTA Strategy

- Primary: "Check room package."
- Secondary: "Continue to tickets."
- Detail link: "Package details and restrictions."

### Operational Assumptions

- Event detail pages can receive package content for eligible shows.
- A package detail or booking destination exists.
- Terms can be shown without building new disclosure UI.

### Measurement Intent

Track event detail view to package card impression, inclusion expansion, package CTA click, ticket-only click, and booking start.

### Risks

- If ticketing is handled by a third party, downstream attribution may be incomplete.
- Package card needs disciplined copy to avoid crowding the event detail page.

## Concept 4: Complete The Night Related Rail

### Summary

Repurpose an existing related content rail/grid to recommend dining, nightlife, room upgrade, and VIP options tied to the event context.

### Target Guest

Guests who are planning an occasion around a show and need help connecting dinner, stay, and post-show plans.

### Package Promise

The page makes the night feel easy to plan without requiring a full bundle checkout.

### Existing Components

- Related content rail or grid.
- Existing restaurant/nightlife/room cards.
- Existing image, title, description, and CTA pattern.

### Placement

Below event details or below the event listing as a supporting module.

### CTA Strategy

- Card CTAs: "Reserve dinner," "Explore rooms," "Inquire about VIP," "View nightlife."
- Optional rail heading: "Complete the night."

### Operational Assumptions

- Recommendations can be curated manually by event, venue, or campaign.
- Dining and VIP destinations already exist.
- No guarantee language is used unless confirmed operationally.

### Measurement Intent

Track related card impressions, cross-sell clicks, reservation starts, room exploration, VIP inquiries, and attachment behavior.

### Risks

- Cross-sell can feel generic if not tied to event timing or occasion.
- Too many recommendations can dilute the package message.

## Concept 5: Page-Wide Stay + Show Promo Banner

### Summary

Use an existing promo banner or strip to add lightweight package visibility across Resorts World Live.

### Target Guest

All Live page visitors, especially mobile users who may not reach lower-page package cards.

### Package Promise

Room + ticket packages are available for select shows, with a direct path to learn more.

### Existing Components

- Promo banner or alert strip.
- Existing CTA link.
- Existing campaign copy style.

### Placement

Below the hero, above event listings, or as a campaign strip between major page sections.

### CTA Strategy

- Primary link: "Explore room + ticket packages."
- Keep banner copy short and non-disruptive.

### Operational Assumptions

- A package landing destination or filtered offer page exists.
- Banner can be configured without layout work.
- It can be hidden when no packages are active.

### Measurement Intent

Track banner impression, CTA click, package landing arrival, and engagement by device.

### Risks

- Broad messaging may be less relevant than event-specific merchandising.
- It can become easy to ignore if styled like a generic promo.
