# Generated Concepts

This file will hold generated concepts after the current-state audit, opportunity framing, package strategy, and telemetry needs are clear. Each concept should be structured enough for scoring, design prompting, and stakeholder review.

## Concept Generation Rules

Each concept should define:

- Working title.
- Target guest segment.
- Occasion or trip context.
- Core package promise.
- Included and optional elements.
- Operational constraints or assumptions.
- Primary CTA and secondary actions.
- Journey sequence.
- Component candidates.
- Measurement intent.
- Risks and open questions.
