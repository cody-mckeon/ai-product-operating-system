# Package Experience Generation

## 1. Workflow Philosophy

Package experience generation is an operational AI workflow for turning hospitality offers into coherent, conversion-aware landing page concepts. It is designed for rapid UX ideation without sacrificing brand integrity, storytelling discipline, or system reuse.

The workflow treats AI-assisted design tools as collaborative concept engines, not random layout machines. Figma Make, Claude, ChatGPT, v0, Aura, and future OpenClaw orchestration should be used to generate structured directions, compare options, pressure-test UX decisions, and produce modular page systems that can be refined by designers, product leads, marketers, and operators.

The goal is cinematic hospitality with operational clarity: immersive enough to create desire, restrained enough to feel premium, and structured enough to scale across packages, seasons, venues, and guest segments.

## 2. AI Generation Objectives

Use this workflow to generate package landing page concepts that:

- Create rapid UX directions for new hospitality, entertainment, culinary, wellness, and premium experiences.
- Translate package value into modular storytelling sections.
- Support mobile-first browsing, scanning, comparison, and booking intent.
- Encourage reusable components instead of one-off page inventions.
- Balance emotional appeal with practical decision support.
- Explore conversion strategies without aggressive sales pressure.
- Produce clear design prompts for AI layout tools and human review.
- Enable repeatable experimentation across narrative, offer, visual, and CTA variants.

AI output should help teams move from vague package idea to reviewable concept quickly. It should not replace brand judgment, content strategy, accessibility review, or product validation.

## 3. Input Context Requirements

Every generation request should include enough context for the AI system to understand the experience, audience, offer mechanics, and design constraints.

Required input:

- Package name or working title.
- Property, venue, restaurant, spa, event, or destination context.
- Primary guest segment.
- Core included benefits.
- Booking window, stay window, or campaign timing.
- Price framing, discount framing, or premium positioning.
- Operational constraints such as blackout dates, capacity, inventory, age limits, or redemption rules.
- Primary conversion action.
- Desired emotional posture.
- Existing page, component, or brand references if available.

Optional input:

- Competitive references.
- Photography direction.
- SEO priorities.
- Personalization signals.
- Analytics learnings.
- A/B test hypotheses.
- Localization requirements.
- Accessibility concerns.

## 4. Required Brand Context

Before generating layouts, provide brand context that anchors the concept.

Brand context should define:

- Tone: cinematic, refined, warm, calm, useful, and confident.
- Visual posture: immersive hospitality over generic product marketing.
- Luxury level: calm luxury instead of loud exclusivity.
- Guest relationship: invitational and attentive, never pushy.
- Content posture: concise, sensory, specific, and operationally clear.
- Trust posture: transparent inclusions, limitations, and booking expectations.

Generation rule:

The AI must preserve the feeling of hospitality. A package page should feel like an invitation into an experience, not a SaaS funnel, casino promotion wall, or discount landing page.

## 5. Required Component Context

Provide the available component system before asking an AI tool to generate a page. If the component system is incomplete, instruct the AI to propose reusable modules rather than inventing disconnected sections.

Component context should include:

- Hero or immersive lead section patterns.
- Package summary modules.
- Benefit and inclusion lists.
- Date, availability, and pricing modules.
- Itinerary or experience sequence modules.
- Venue, room, restaurant, event, or spa cards.
- Editorial image and caption modules.
- Testimonial, review, or social proof modules.
- FAQ and policy modules.
- Sticky mobile CTA patterns.
- Related package modules.
- Comparison or upgrade modules.

Generation rule:

Each generated section must map to a reusable module name. If a section cannot be named as a reusable module, the AI should simplify it until it can.

## 6. Required Token Context

Provide token context whenever available so AI-generated concepts stay aligned with the design system.

Token context should include:

- Color roles, not only raw colors.
- Typography scale and editorial heading rules.
- Spacing scale.
- Border radius principles.
- Elevation and shadow rules.
- Motion principles.
- Image aspect ratios.
- Breakpoint behavior.
- Button and CTA variants.
- Form, selector, tab, and accordion states.

Generation rule:

AI tools may suggest token usage, but should not invent a new visual system unless explicitly asked to explore a future direction. Concepts should use token roles such as `surface`, `foreground`, `accent`, `muted`, `border`, `cta`, and `success` where relevant.

## 7. UX Generation Rules

Generated concepts must follow these rules:

- Start with guest motivation, not internal business structure.
- Make the offer understandable within the first screen.
- Build desire through atmosphere, sequence, and specificity.
- Keep inclusions and rules easy to scan.
- Use progressive disclosure for details that are necessary but not emotionally central.
- Avoid unrelated modules, visual clutter, and excessive promotional badges.
- Make every section earn its place in the package decision journey.
- Support both exploratory browsing and high-intent booking.
- Prefer clear section rhythm over novelty.
- Preserve accessibility, readable contrast, and touch-friendly controls.

The AI should generate fewer, better sections rather than filling a page with generic content blocks.

## 8. Hospitality Experience Framing

Hospitality package pages should frame the offer as an experience with a beginning, emotional center, and practical path to booking.

Use the following framing layers:

- Arrival: what the guest sees, feels, or anticipates first.
- Access: what the package unlocks that ordinary booking does not.
- Ritual: the signature sequence of moments.
- Ease: how the package removes planning friction.
- Belonging: who the experience is for and why it fits them.
- Memory: what the guest will remember after the stay or event.
- Clarity: what is included, what is not, and how to reserve.

Generation rule:

Every concept should identify its primary hospitality promise in one sentence before generating layout.

## 9. Package Narrative Structures

Choose one narrative structure before generating the page.

### The Cinematic Arrival

Best for premium stays, destination launches, and seasonal campaigns.

Structure:

- Immersive hero with offer promise.
- Sensory editorial section.
- Package inclusions.
- Experience sequence.
- Room, venue, or amenity highlight.
- Booking clarity.
- Related experiences.

### The Effortless Escape

Best for wellness, couples, family, and weekend packages.

Structure:

- Calm hero focused on relief or ease.
- What is handled for the guest.
- Day-by-day or moment-by-moment flow.
- Practical inclusions.
- Trust and policy details.
- Soft CTA.

### The Insider Access

Best for entertainment, event, and room + ticket packages.

Structure:

- Access-led hero.
- Ticket, seat, or event value.
- Stay or dining pairing.
- Timeline for the night.
- Upgrade paths.
- Availability and urgency.
- Direct booking CTA.

### The Curated Indulgence

Best for culinary, spa, premium upsell, and luxury packages.

Structure:

- Editorial hero with taste, touch, or atmosphere.
- Signature inclusions.
- Craft or expert-led detail.
- Optional enhancements.
- Social proof or authority.
- Booking path.

## 10. Emotional Positioning Variants

Generate emotional variants to compare how the same package can be framed.

Recommended variants:

- Calm Luxury: quiet confidence, spacious layout, restrained copy, premium details.
- Immersive Discovery: layered imagery, exploratory modules, destination-led storytelling.
- Effortless Planning: practical clarity, bundled convenience, low-friction booking.
- Celebration: warm energy, social moments, special occasion cues.
- Insider Access: exclusive availability, privileged entry, event proximity.
- Restoration: wellness, softness, decompression, renewal.
- Culinary Anticipation: sensory specificity, chef or menu detail, appetite-driven pacing.

Each variant should change narrative emphasis and section priority. It should not merely swap adjectives.

## 11. Conversion Strategy Variants

Choose a conversion strategy that matches the package intent.

Recommended variants:

- Direct Booking: strongest for high-intent offers with clear inventory.
- Explore Then Commit: strongest for complex or premium experiences that need storytelling.
- Compare Packages: strongest when multiple bundles compete for the same audience.
- Upgrade Path: strongest when the package sits above standard room booking.
- Limited Window: strongest for seasonal or event-driven campaigns.
- Concierge Inquiry: strongest for luxury, group, high-price, or custom experiences.
- Save and Return: strongest for considered travel planning across devices.

Generation rule:

Conversion pressure should feel appropriate to the guest decision. Avoid repetitive CTAs, countdown clutter, false urgency, and aggressive discount framing.

## 12. Mobile-First Generation Rules

All package concepts must be generated mobile-first before desktop expansion.

Mobile rules:

- The first viewport must communicate package name, primary promise, key inclusion, and CTA.
- Use a sticky bottom CTA only when it improves booking clarity.
- Keep hero copy concise and readable over imagery.
- Avoid dense grids that collapse into long, repetitive stacks.
- Place package inclusions before deep editorial content when booking intent is likely high.
- Use accordions for policies, terms, and dense details.
- Keep comparison modules short and scannable.
- Use image aspect ratios that preserve subject focus on narrow screens.
- Ensure tap targets are comfortable and obvious.
- Avoid placing critical information only in hover states.

Desktop should expand the story, not introduce a different content strategy.

## 13. Editorial Layout Guidance

Editorial layouts should create atmosphere while preserving decision clarity.

Guidance:

- Use large imagery only when it carries specific hospitality meaning.
- Pair cinematic visuals with concise, concrete copy.
- Alternate sensory storytelling with practical modules.
- Use captions for operationally useful context, not decorative filler.
- Let whitespace signal calm luxury.
- Avoid magazine-like layouts that hide the offer mechanics.
- Use editorial rhythm to reveal the package sequence.
- Keep section transitions coherent: arrival, inclusion, experience, proof, booking.

The page should feel curated, not assembled.

## 14. Visual Hierarchy Guidance

The hierarchy should guide the guest from desire to clarity to action.

Priority order:

1. Package promise.
2. Primary experience or access.
3. Key inclusions.
4. Booking action.
5. Dates, price, availability, and restrictions.
6. Supporting story and proof.
7. Policies and FAQs.
8. Related packages or upgrades.

Visual hierarchy rules:

- Do not let badges, discounts, or secondary CTAs overpower the experience promise.
- Use one primary CTA per decision moment.
- Keep pricing and eligibility visible when they materially affect conversion.
- Use section headings that describe guest value, not internal content types.
- Keep repeated modules visually consistent across concepts.

## 15. CTA Placement Guidance

CTA placement should support intent without creating pressure.

Recommended placements:

- Primary CTA in the hero.
- Secondary detail link near inclusions when terms are important.
- Sticky mobile CTA for booking, checking availability, or inquiry.
- Mid-page CTA after the package value is clear.
- Final CTA after FAQs or booking details.
- Upgrade CTA only near relevant enhancement content.

CTA language should be specific:

- `Check Availability`
- `Reserve This Package`
- `Explore Dates`
- `Plan This Stay`
- `Request Concierge Support`
- `Add This Experience`

Avoid vague or noisy CTA language such as `Learn More`, `Act Now`, `Claim Deal`, or repeated urgency phrasing unless the campaign strategy explicitly requires it.

## 16. AI Iteration Strategy

Use AI iteration in controlled passes.

Recommended sequence:

1. Clarify the package promise and audience.
2. Generate three narrative directions.
3. Select one narrative structure and one emotional variant.
4. Generate mobile-first wireframe sections.
5. Expand into desktop layout.
6. Generate component mapping.
7. Generate copy alternatives for hero, inclusions, and CTAs.
8. Generate conversion variants.
9. Compare concepts using the comparison framework.
10. Refine the strongest concept against brand, component, token, and governance rules.

Iteration rule:

Each pass should have a clear instruction and evaluation target. Do not ask AI tools to "make it better" without defining what better means.

## 17. Concept Comparison Framework

Compare generated concepts using consistent criteria.

Score each concept from 1 to 5:

- Brand fit: Does it feel cinematic, refined, warm, and operationally useful?
- Package clarity: Can the guest understand what is included?
- Emotional pull: Does the concept create desire without exaggeration?
- Mobile usability: Does the concept work as a first-class mobile experience?
- Conversion quality: Does the CTA strategy match guest intent?
- Reusability: Can the sections become reusable package modules?
- Operational accuracy: Are constraints and rules visible enough?
- Experiment value: Does the concept teach something testable?
- Accessibility posture: Are readability, contrast, and interaction states considered?
- Content scalability: Can the structure support future packages?

Preferred concepts should score highly on clarity, brand fit, mobile usability, and reusability before visual novelty.

## 18. Telemetry-Aware UX Thinking

AI-generated concepts should anticipate measurement without letting telemetry flatten the experience.

Consider:

- Hero CTA engagement.
- Inclusion module expansion.
- Date or availability interaction.
- Package comparison usage.
- Scroll depth by section.
- FAQ expansion.
- Sticky CTA engagement.
- Image gallery engagement.
- Upgrade module selection.
- Booking start and booking completion.
- Inquiry submission.
- Return visits or save actions.

Telemetry questions:

- Where might guests hesitate?
- Which module should prove value fastest?
- Which details reduce booking anxiety?
- Which section should be personalized in a future iteration?
- Which CTA placement should be tested?

Generation rule:

Every concept should identify three events or signals that would help evaluate whether the UX is working.

## 19. Experimentation Guidance

Use the workflow to create scalable experiments, not isolated page comps.

Experiment dimensions:

- Narrative: cinematic arrival vs effortless escape vs insider access.
- Offer framing: included value vs exclusive access vs convenience.
- CTA strategy: booking, availability, inquiry, upgrade, save.
- Module order: inclusions early vs story early.
- Proof: guest review, expert authority, venue reputation, partner credibility.
- Visual density: immersive editorial vs concise booking utility.
- Pricing posture: premium anchor vs value clarity.
- Personalization: segment-specific hero, inclusions, or recommendations.

Experiment rule:

Change one primary strategic dimension at a time when generating test concepts. Multiple visual changes are acceptable only when they support the same hypothesis.

## 20. Future Agent Orchestration

Future OpenClaw orchestration should treat this workflow as a multi-agent design operation.

Recommended agent roles:

- Context Agent: gathers package, brand, component, token, and operational inputs.
- Strategy Agent: chooses narrative, emotional, and conversion variants.
- Layout Agent: generates mobile-first and desktop section structures.
- Copy Agent: writes hospitality-specific copy and CTA alternatives.
- Systems Agent: maps sections to reusable components and token roles.
- Telemetry Agent: proposes measurement events and experiment hypotheses.
- Critique Agent: reviews against brand, accessibility, clarity, and governance.
- Handoff Agent: formats outputs for Figma Make, Claude, ChatGPT, v0, Aura, or implementation teams.

Orchestration rule:

Agents should pass structured artifacts forward. The workflow should avoid hidden reasoning chains and instead produce explicit decisions, assumptions, constraints, and review notes.

- Include enough structure for direct use in Figma Make, Claude, ChatGPT, v0, Aura, or orchestration.

## 22. Review and Governance Process

AI-generated concepts require human review before production use.

Reviewers should check:

- Brand alignment.
- Offer accuracy.
- Operational feasibility.
- Legal, policy, and restriction clarity.
- Accessibility and readability.
- Mobile usability.
- Component reusability.
- Token alignment.
- Measurement readiness.
- Experiment validity.

Governance rules:

- Do not ship AI-generated package pages without validating offer mechanics.
- Do not introduce new component patterns without documenting reuse potential.
- Do not use AI-generated imagery, copy, or claims without rights and accuracy review.
- Do not prioritize conversion tactics that damage trust.
- Archive rejected concepts with notes when they produce useful learning.

## Tool-Specific Usage Notes

### Figma Make

Use Figma Make for visual layout exploration, responsive section rhythm, and early component ideas. Provide structured component context, token context, and mobile-first rules before asking for generation.

### Claude

Use Claude for narrative development, critique, governance checks, and long-form concept comparison. Provide the concept output format and ask for explicit assumptions and risks.

### ChatGPT

Use ChatGPT for strategy variants, prompt refinement, copy options, orchestration planning, and cross-tool handoff formatting.

### v0

Use v0 for implementation-adjacent layout exploration when the team wants to understand how a concept might translate into a componentized front-end system. Keep prompts tied to existing components and tokens.

### Aura

Use Aura for visual atmosphere, hospitality mood exploration, campaign direction, and image-led ideation when brand and rights review will follow.

### Future OpenClaw Orchestration

Use OpenClaw to coordinate multi-agent generation, structured review, experiment setup, and reusable artifact creation across tools.
